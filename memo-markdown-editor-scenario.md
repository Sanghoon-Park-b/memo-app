# E2E 테스트 시나리오: 마크다운 에디터 기능

## 목적

마크다운 에디터가 올바르게 작동하고 콘텐츠가 정상적으로 렌더링되는지 확인

## 시나리오 단계

### 마크다운 에디터 기본 기능

1. 브라우저에서 http://localhost:3000 접속
2. "새 메모" 버튼 클릭
3. 마크다운 에디터 영역이 표시되는지 확인
4. 에디터 높이가 적절한지 확인 (400px)

### 기본 마크다운 문법 입력

5. 다음 마크다운 텍스트를 에디터에 입력:
   ```
   # 제목 1
   ## 제목 2
   ### 제목 3
   
   **굵은 글씨**
   *기울임 글씨*
   
   - 리스트 항목 1
   - 리스트 항목 2
   - 리스트 항목 3
   
   1. 번호 목록 1
   2. 번호 목록 2
   
   `코드 블록`
   
   [링크](https://example.com)
   
   > 인용문
   ```

### 메모 저장 및 렌더링 확인

6. 제목에 "마크다운 테스트" 입력
7. "저장하기" 버튼 클릭
8. 저장된 메모 카드 클릭하여 상세 모달 열기
9. 마크다운이 올바르게 HTML로 렌더링되어 표시되는지 확인:
   - 제목들이 적절한 크기로 렌더링
   - 굵은 글씨와 기울임 글씨 적용
   - 리스트가 올바른 형식으로 표시
   - 번호 목록이 순서대로 표시
   - 코드 블록이 하이라이트되어 표시
   - 링크가 클릭 가능한 상태로 표시
   - 인용문이 적절한 스타일로 표시

### 편집 모드에서 마크다운 확인

10. 상세 모달에서 편집 버튼 클릭
11. 편집 모달에서 기존 마크다운 내용이 원본 그대로 표시되는지 확인
12. 내용 일부 수정 (예: 새로운 문단 추가)
13. "수정하기" 버튼 클릭
14. 다시 상세 모달에서 수정된 내용이 올바르게 렌더링되는지 확인

### 복잡한 마크다운 구조 테스트

15. 새 메모 작성
16. 더 복잡한 마크다운 구조 입력:
   ```
   # 복잡한 마크다운 테스트
   
   ## 코드 블록
   ```javascript
   function hello() {
       console.log("Hello, World!");
   }
   ```
   
   ## 표
   | 항목 | 값 |
   |------|-----|
   | 이름 | 홍길동 |
   | 나이 | 30 |
   
   ## 체크리스트
   - [x] 완료된 항목
   - [ ] 미완료 항목
   
   ## 수평선
   ---
   
   텍스트가 여기에 있습니다.
   ```

17. 저장 후 렌더링 결과 확인
18. 코드 블록이 syntax highlighting과 함께 표시되는지 확인
19. 표가 올바른 형식으로 렌더링되는지 확인
20. 체크리스트가 적절히 표시되는지 확인
21. 수평선이 렌더링되는지 확인

### 에디터 UI 테스트

22. 새 메모 작성 모달 열기
23. 마크다운 에디터의 툴바 존재 여부 확인
24. 에디터 영역의 포커스 상태 확인
25. 텍스트 입력 시 즉시 반영되는지 확인

### 긴 콘텐츠 처리

26. 매우 긴 마크다운 콘텐츠 입력 (여러 섹션과 긴 문단)
27. 에디터가 스크롤 가능한지 확인
28. 저장 후 상세 모달에서 긴 콘텐츠가 올바르게 표시되는지 확인
29. 모달 내에서 스크롤이 정상 작동하는지 확인

## 예상 결과

- 마크다운 에디터가 정상적으로 로드되고 작동함
- 입력된 마크다운이 올바른 HTML로 렌더링됨
- 복잡한 마크다운 구조도 정확히 처리됨
- 편집 시 원본 마크다운이 보존됨
- 긴 콘텐츠도 적절히 처리됨
