# E2E 테스트 시나리오: 폼 유효성 검사

## 목적

메모 작성 시 필수 항목들이 올바르게 검증되는지 확인

## 시나리오 단계

### 빈 제목으로 저장 시도

1. 브라우저에서 http://localhost:3000 접속
2. "새 메모" 버튼 클릭
3. 제목 입력란을 비워둔 채로 내용만 입력
4. "저장하기" 버튼 클릭
5. "제목과 내용을 모두 입력해주세요." 알림 메시지 확인
6. 모달이 닫히지 않고 그대로 유지되는지 확인

### 빈 내용으로 저장 시도

7. 제목은 입력하고 내용 입력란을 비워둔 채로 저장 시도
8. "저장하기" 버튼 클릭
9. "제목과 내용을 모두 입력해주세요." 알림 메시지 확인
10. 모달이 닫히지 않고 그대로 유지되는지 확인

### 제목과 내용 모두 비워둔 채로 저장 시도

11. 제목과 내용을 모두 비워둔 채로 저장 시도
12. "저장하기" 버튼 클릭
13. "제목과 내용을 모두 입력해주세요." 알림 메시지 확인

### 공백만 입력된 필드 검증

14. 제목 입력란에 공백만 입력 (스페이스바 여러 번)
15. 내용에는 정상적인 텍스트 입력
16. "저장하기" 버튼 클릭
17. 공백 제목이 유효성 검사에 걸리는지 확인

18. 제목에는 정상 텍스트, 내용에는 공백만 입력
19. "저장하기" 버튼 클릭
20. 공백 내용이 유효성 검사에 걸리는지 확인

### 편집 시 유효성 검사

21. 기존 메모 편집 시작
22. 제목을 완전히 삭제
23. "수정하기" 버튼 클릭
24. 동일한 유효성 검사 메시지가 표시되는지 확인

### 태그 유효성 (선택사항)

25. 제목과 내용을 올바르게 입력
26. 태그 입력란에 빈 문자열이나 공백 입력 후 "추가" 버튼 클릭
27. 빈 태그가 추가되지 않는지 확인

### 중복 태그 방지

28. 태그 "테스트" 추가
29. 동일한 태그 "테스트" 다시 추가 시도
30. 중복 태그가 추가되지 않는지 확인

### 성공적인 저장

31. 모든 필수 필드를 올바르게 입력
32. "저장하기" 버튼 클릭
33. 유효성 검사를 통과하고 메모가 정상 저장되는지 확인
34. 모달이 닫히고 목록에 새 메모가 표시되는지 확인

## 예상 결과

- 필수 필드 누락 시 적절한 오류 메시지 표시
- 공백만 있는 필드도 빈 필드로 처리됨
- 유효성 검사 실패 시 폼이 닫히지 않음
- 중복 태그와 빈 태그가 추가되지 않음
- 모든 유효성 검사를 통과해야만 저장됨
