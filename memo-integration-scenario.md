# E2E 테스트 시나리오: 통합 사용자 워크플로우

## 목적

실제 사용자가 수행할 만한 전체적인 워크플로우를 통해 모든 기능이 함께 잘 작동하는지 확인

## 시나리오 단계

### 초기 상태 확인 및 첫 메모 작성

1. 브라우저에서 http://localhost:3000 접속
2. 빈 메모 목록 상태 확인 ("아직 메모가 없습니다" 메시지)
3. "새 메모" 버튼 클릭
4. 첫 번째 메모 작성:
   - 제목: "프로젝트 아이디어 모음"
   - 카테고리: "아이디어"
   - 내용: "# 새로운 앱 아이디어\n\n- 할일 관리 앱\n- 독서 기록 앱\n- 운동 추적 앱"
   - 태그: "아이디어", "앱"
5. 저장 후 메모 목록에 표시 확인

### 여러 카테고리의 메모들 추가 작성

6. 두 번째 메모 (업무):
   - 제목: "주간 회의 내용"
   - 카테고리: "업무"
   - 내용: "## 이번 주 목표\n\n**완료해야 할 일:**\n1. API 문서 작성\n2. 코드 리뷰\n3. 테스트 케이스 추가"
   - 태그: "회의", "업무"

7. 세 번째 메모 (학습):
   - 제목: "React 학습 노트"
   - 카테고리: "학습"
   - 내용: "useState와 useEffect 훅 정리\n\n`useState`는 상태 관리를 위해 사용"
   - 태그: "React", "JavaScript"

8. 네 번째 메모 (개인):
   - 제목: "오늘의 일기"
   - 카테고리: "개인"
   - 내용: "오늘은 날씨가 좋았다. 산책을 다녀왔다."
   - 태그: "일기"

### 검색 및 필터링 워크플로우

9. 총 4개의 메모가 목록에 표시되는지 확인
10. 검색창에 "앱" 입력하여 첫 번째 메모만 필터링 확인
11. 검색 초기화
12. 카테고리 필터에서 "업무" 선택하여 업무 메모만 표시 확인
13. 필터 초기화하여 전체 목록 복원

### 메모 상세 조회 및 편집

14. "프로젝트 아이디어 모음" 메모 카드 클릭
15. 상세 모달에서 마크다운 렌더링 확인
16. 편집 버튼 클릭
17. 내용에 새로운 아이디어 추가: "- 날씨 앱"
18. 새 태그 "모바일" 추가
19. 수정 저장 후 변경사항 반영 확인

### 메모 간 이동 및 관리

20. 다른 메모("주간 회의 내용") 카드 클릭하여 상세 보기
21. 해당 메모에서 직접 편집하여 완료 항목 체크:
    ```
    1. ~~API 문서 작성~~ ✅
    2. 코드 리뷰
    3. 테스트 케이스 추가
    ```
22. 저장 후 목록으로 돌아가기

### 카테고리별 통계 및 필터링 재확인

23. 카테고리 드롭다운에서 각 카테고리의 메모 개수 확인:
    - 아이디어 (1), 업무 (1), 학습 (1), 개인 (1)
24. "학습" 카테고리 선택하여 React 노트만 표시
25. 해당 메모의 편집 버튼으로 추가 학습 내용 작성

### 검색 조합 테스트

26. 필터 초기화
27. 검색창에 "React" 입력하여 학습 메모 필터링
28. 동시에 카테고리를 "학습"으로 설정
29. 검색과 카테고리 필터가 함께 작동하는지 확인
30. 통계 정보가 올바르게 표시되는지 확인

### 메모 삭제 및 목록 관리

31. 필터 초기화하여 전체 메모 목록 표시
32. "오늘의 일기" 메모 삭제
33. 총 3개 메모로 줄어든 것 확인
34. 카테고리 통계에서 "개인" 카테고리가 0개로 변경된 것 확인

### 태그 기반 검색

35. 검색창에 "JavaScript" 입력
36. React 학습 노트가 검색되는지 확인 (태그 기반 검색)
37. 검색창에 "앱" 입력
38. 프로젝트 아이디어 메모가 검색되는지 확인

### 대량 콘텐츠 처리

39. 새 메모 작성으로 긴 콘텐츠 메모 생성:
    - 제목: "상세한 기술 문서"
    - 여러 섹션과 긴 내용이 포함된 마크다운
40. 저장 후 메모 카드에서 내용이 적절히 축약 표시되는지 확인
41. 상세 모달에서 긴 내용이 스크롤과 함께 잘 표시되는지 확인

### 에러 상황 처리

42. 새 메모 작성 시 제목만 입력하고 내용 비워둔 채 저장 시도
43. 유효성 검사 메시지 확인
44. 올바르게 입력하여 저장 성공

### 최종 상태 확인

45. 총 4개의 메모가 있는지 확인
46. 각 카테고리별 분포 확인
47. 검색 기능이 여전히 정상 작동하는지 확인
48. 모든 메모의 편집/삭제 기능이 정상인지 확인

### 브라우저 새로고침 후 데이터 지속성

49. 브라우저 새로고침 (F5)
50. 모든 메모가 그대로 유지되는지 확인
51. 검색과 필터 상태가 초기화되었는지 확인
52. 모든 기능이 여전히 정상 작동하는지 확인

## 예상 결과

- 전체 사용자 워크플로우가 자연스럽게 연결됨
- 모든 CRUD 기능이 일관되게 작동
- 검색과 필터링이 조합되어 잘 작동
- 데이터 무결성이 유지됨
- UI/UX가 직관적이고 사용하기 쉬움
- 브라우저 새로고침 후에도 데이터가 유지됨
