# E2E 테스트 시나리오: 메모 삭제 기능

## 목적

메모를 안전하게 삭제할 수 있고 확인 절차가 올바르게 작동하는지 확인

## 시나리오 단계

### 메모 목록에서 삭제 - 확인

1. 브라우저에서 http://localhost:3000 접속
2. 메모 목록이 로드될 때까지 대기
3. 삭제할 메모의 총 개수 확인
4. 첫 번째 메모 카드의 삭제 버튼(휴지통 아이콘) 클릭
5. 확인 다이얼로그 "정말로 이 메모를 삭제하시겠습니까?" 표시 확인
6. "확인" 버튼 클릭
7. 해당 메모가 목록에서 제거되었는지 확인
8. 메모 총 개수가 1개 줄어들었는지 확인

### 메모 목록에서 삭제 - 취소

9. 다른 메모의 삭제 버튼 클릭
10. 확인 다이얼로그 표시 확인
11. "취소" 버튼 클릭
12. 메모가 삭제되지 않고 목록에 그대로 있는지 확인

### 상세 모달에서 삭제 - 확인

13. 메모 카드 클릭하여 상세 모달 열기
14. 모달 내 삭제 버튼(휴지통 아이콘) 클릭
15. 확인 다이얼로그 표시 확인
16. "확인" 버튼 클릭
17. 모달이 자동으로 닫히는지 확인
18. 해당 메모가 목록에서 제거되었는지 확인

### 상세 모달에서 삭제 - 취소

19. 다른 메모 카드 클릭하여 상세 모달 열기
20. 모달 내 삭제 버튼 클릭
21. 확인 다이얼로그에서 "취소" 선택
22. 모달이 그대로 열려있는지 확인
23. 메모가 삭제되지 않았는지 확인

### 필터링된 상태에서 삭제

24. 검색어 입력하여 메모 필터링
25. 필터된 메모 중 하나 삭제
26. 필터링된 목록에서 해당 메모가 제거되는지 확인
27. 필터 초기화 후 전체 목록에서도 제거되었는지 확인

### 카테고리별 통계 업데이트

28. 특정 카테고리의 메모 개수 확인
29. 해당 카테고리의 메모 삭제
30. 카테고리 드롭다운에서 해당 카테고리의 메모 개수가 줄어들었는지 확인

### 마지막 메모 삭제

31. 모든 메모를 삭제하여 목록이 비워질 때까지 반복
32. "아직 메모가 없습니다" 메시지 표시 확인
33. "첫 번째 메모를 작성해보세요!" 안내 메시지 확인

## 예상 결과

- 삭제 확인 다이얼로그가 항상 표시됨
- 확인 시에만 메모가 삭제됨
- 취소 시 메모가 보존됨
- 삭제 후 UI가 즉시 업데이트됨
- 통계 정보가 정확히 반영됨
- 빈 목록 상태의 적절한 안내 메시지 표시
