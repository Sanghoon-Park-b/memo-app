# E2E 테스트 시나리오: 반응형 UI 및 사용성

## 목적

애플리케이션이 다양한 화면 크기에서 올바르게 작동하고 사용자 경험이 일관적인지 확인

## 시나리오 단계

### 데스크톱 뷰 (1920x1080)

1. 브라우저를 1920x1080 해상도로 설정
2. http://localhost:3000 접속
3. 헤더 레이아웃이 올바르게 표시되는지 확인
4. 메모 목록이 3열 그리드로 표시되는지 확인 (lg:grid-cols-3)
5. 검색바와 카테고리 필터가 한 줄에 표시되는지 확인
6. "새 메모" 버튼이 우상단에 적절히 배치되는지 확인

### 태블릿 뷰 (768x1024)

7. 브라우저를 768x1024 해상도로 조정
8. 메모 목록이 2열 그리드로 변경되는지 확인 (md:grid-cols-2)
9. 검색바와 카테고리 필터 레이아웃 확인
10. 모든 버튼과 인터페이스 요소가 터치 친화적인 크기인지 확인

### 모바일 뷰 (375x667)

11. 브라우저를 375x667 해상도로 조정 (iPhone SE 크기)
12. 메모 목록이 1열로 표시되는지 확인
13. 검색바와 카테고리 필터가 세로로 배열되는지 확인 (flex-col sm:flex-row)
14. 헤더의 제목과 버튼이 적절히 배치되는지 확인
15. 메모 카드의 내용이 모바일 화면에 맞게 조정되는지 확인

### 모달 반응형 테스트

16. 모바일 뷰에서 "새 메모" 버튼 클릭
17. 메모 작성 모달이 화면에 맞게 표시되는지 확인
18. 모달이 전체 화면을 적절히 활용하는지 확인 (max-w-2xl w-full)
19. 마크다운 에디터가 모바일에서 사용 가능한지 확인
20. 버튼들이 모바일에서 누르기 쉬운 크기인지 확인

### 태블릿에서 모달 테스트

21. 태블릿 뷰로 전환
22. 메모 상세 보기 모달 열기
23. 모달 크기가 태블릿 화면에 적합한지 확인
24. 내용이 적절히 표시되는지 확인

### 터치 인터랙션 테스트

25. 모바일/태블릿 뷰에서 메모 카드 터치
26. 호버 효과 대신 터치 피드백이 작동하는지 확인
27. 편집/삭제 버튼이 터치하기 쉬운 크기인지 확인
28. 태그 제거 버튼이 작은 화면에서도 조작 가능한지 확인

### 스크롤 및 네비게이션

29. 모바일에서 긴 메모 목록 스크롤 테스트
30. 모달 내에서 긴 콘텐츠 스크롤 테스트
31. 검색 결과가 많을 때 스크롤 동작 확인

### 가로 방향 테스트

32. 모바일을 가로 방향으로 회전 (667x375)
33. 레이아웃이 적절히 조정되는지 확인
34. 모달이 가로 화면에서도 잘 작동하는지 확인

### 작은 화면에서의 텍스트 가독성

35. 다양한 화면 크기에서 폰트 크기 확인
36. 메모 제목과 내용이 읽기 쉬운지 확인
37. 버튼 텍스트가 잘리지 않는지 확인

### 극단적인 화면 크기

38. 매우 좁은 화면 (320px)에서 테스트
39. 레이아웃이 깨지지 않는지 확인
40. 모든 기능이 여전히 접근 가능한지 확인

41. 매우 큰 화면 (2560px 이상)에서 테스트
42. 컨텐츠가 적절히 중앙 정렬되는지 확인 (max-w-7xl mx-auto)
43. 메모 카드들이 과도하게 늘어나지 않는지 확인

## 예상 결과

- 모든 화면 크기에서 레이아웃이 적절히 조정됨
- 터치 친화적인 인터페이스 제공
- 텍스트와 버튼이 모든 화면에서 읽기 쉽고 조작 가능
- 모달이 다양한 화면 크기에 적응함
- 극단적인 화면 크기에서도 기능이 정상 작동
- 사용자 경험이 모든 디바이스에서 일관됨
